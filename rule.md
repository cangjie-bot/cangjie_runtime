# 添加示例代码的原则：

- 尽量使用仓颉std库来帮助实现
- 代码编写完后，请使用cjc xxx.cj 进行编译验证，并通过 ./main 的方式进行执行。
- 执行的结果要和文档中“运行结果”完全一致
- 示例代码要简洁，要使用了目标函数
- md文档的增删改查你只能增和查，但是不能删和改。
- 只针对每一个函数（不包括纯接口interface，但是包括init函数）进行添加，如果标题是interface就不用管他
- deprecated 标题的函数可以不用加示例代码
- 如果示例代码的结果是随机的，那么示例代码的标签就从 verify 改成 run。<\!-- run -->
- 写的示例代码尽量不要带 warning
- 重要！不要修改其他文档内容，只能添加！
- 临时创建的 test_xxxx.cj 在用完之后就删除，不要保留。
- 写示例之前，先列一个todolist，经过确认再开始写。
